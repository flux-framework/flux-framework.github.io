# Release Notes

flux-core version 0.84.0 - 2026-04-07
-------------------------------------

## New Features
 * python: add `JobspecV1.apply_options()` and `from_submit`/`alloc`/`batch()`
   factory methods ([#7501](https://github.com/flux-framework/flux-core/issues/7501))
 * broker: support python broker modules ([#7424](https://github.com/flux-framework/flux-core/issues/7424))
 * broker: support module loaders other than the built-in DSO one ([#7422](https://github.com/flux-framework/flux-core/issues/7422))
 * python: add toy scheduler framework ([#7451](https://github.com/flux-framework/flux-core/issues/7451))
 * python: add pure python ResourceSet implementation ([#7457](https://github.com/flux-framework/flux-core/issues/7457))
 * sched-simple: replace with python version ([#7461](https://github.com/flux-framework/flux-core/issues/7461))
 * scheduler: add support for generator yield in schedule/forecast loops
   ([#7507](https://github.com/flux-framework/flux-core/issues/7507))
 * scheduler: support RFC 14 range counts ([#7470](https://github.com/flux-framework/flux-core/issues/7470))
 * python: add `set_proctitle()` and use it in python modules, job validator
   and frobnicator ([#7475](https://github.com/flux-framework/flux-core/issues/7475))
 * python: improve broker module error propagation and test integration
   ([#7447](https://github.com/flux-framework/flux-core/issues/7447))
 * flux-resource: add queue display filtering and hidden-queues config to
   simplify output of `flux resource list` ([#7473](https://github.com/flux-framework/flux-core/issues/7473))
 * flux-update: add `--wait` option and use it as necessary in testsuite
   ([#7446](https://github.com/flux-framework/flux-core/issues/7446))
 * Support `--conf` option in `flux start` and `flux broker` ([#7458](https://github.com/flux-framework/flux-core/issues/7458))
 * enable `flux exec --jobid` in multi-user instance via RFC 42 signing
   ([#7430](https://github.com/flux-framework/flux-core/issues/7430))
 * libidset: define and enforce `IDSET_MAX_UNIVERSE` ([#7497](https://github.com/flux-framework/flux-core/issues/7497))
 * cmd/flux: output similar commands on invalid input ([#7403](https://github.com/flux-framework/flux-core/issues/7403))
 * modprobe: propagate alternatives selection to all module `provides`
   ([#7455](https://github.com/flux-framework/flux-core/issues/7455))
 * update FLUX_CLI_PLUGINPATH to support a colon-delimited path (multiple
   paths) ([#7244](https://github.com/flux-framework/flux-core/issues/7244))

## Fixes
 * sched-simple: fix property and scheduling key propagation in allocations
   ([#7492](https://github.com/flux-framework/flux-core/issues/7492))
 * avoid masking the exit status of the task that triggered an
   `exit-timeout` or `exit-on-error` job exception ([#7474](https://github.com/flux-framework/flux-core/issues/7474))
 * fix resource leaks and flaky tests found when running the testsuite
   under valgrind ([#7445](https://github.com/flux-framework/flux-core/issues/7445))
 * job-info: fix resource leaks ([#7441](https://github.com/flux-framework/flux-core/issues/7441))
 * resource: drop scheduling key from resource.status ([#7469](https://github.com/flux-framework/flux-core/issues/7469))
 * cmd/flux: remove duplicate command suggestions ([#7484](https://github.com/flux-framework/flux-core/issues/7484))
 * job-list: honor job updates in all job states ([#7493](https://github.com/flux-framework/flux-core/issues/7493))
 * python: store `CLIPlugin` options with prefix by default ([#7514](https://github.com/flux-framework/flux-core/issues/7514))

## Cleanup
