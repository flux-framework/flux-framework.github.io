# Release Notes

flux-core version 0.87.0 - 2026-07-07
-------------------------------------

## New Features
 * content: add online garbage collection ([#7681](https://github.com/flux-framework/flux-core/issues/7681))
 * shell: add new `shell.tasks-complete` callback and use it to disable
   exit-timeout ([#7709](https://github.com/flux-framework/flux-core/issues/7709))
 * flux-restore: pipeline content store requests ([#7701](https://github.com/flux-framework/flux-core/issues/7701))
 * job-manager: pipeline KVS lookups when reloading jobs on restart ([#7694](https://github.com/flux-framework/flux-core/issues/7694))
 * speed up KVS traversal in `flux dump` and `flux fsck` by pipelining
   content loads ([#7697](https://github.com/flux-framework/flux-core/issues/7697))
 * job-list: add `t_priority` and `t_sched` attributes ([#7684](https://github.com/flux-framework/flux-core/issues/7684))
 * add `access.private-mode` config key which restricts guest users to their
   own jobs ([#7685](https://github.com/flux-framework/flux-core/issues/7685))
 * flux-fsck: allow online check if --repair is not specified ([#7688](https://github.com/flux-framework/flux-core/issues/7688))
 * add node topology aware scheduling ([#7621](https://github.com/flux-framework/flux-core/issues/7621))
 * python: improve CLI option parsing with `FluxArgumentParser` ([#7675](https://github.com/flux-framework/flux-core/issues/7675))
 * broker: add timeout on built-in module unload ([#7659](https://github.com/flux-framework/flux-core/issues/7659))
